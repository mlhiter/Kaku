## 4. 先前讨论过的成本粗估（仅作初步参考）

1. Pin 标签页：约 2-4 天（低）
2. Codex 会话持久化与恢复：
- MVP 1-2 周
- 完整恢复 3-4 周
3. Sidebar + 项目模型（结构性改造）：约 4-8 周
4. Sidebar 命令/环境变量：约 1-2 周（若含系统密钥链集成）
5. 可编辑 Diff：
- 已有 diff 前提下，MVP 2-4 周
- 稳定化再加 3-6 周
6. GitLens Lite（变更树 + 双栏 diff + hunk 操作 + blame 基础）：约 6-10 周（单人熟练 Rust）
7. 背景图 + 自动文字颜色适配：
- MVP（静态采样 + 全局前景色切换）约 3-5 天
- 增强版（分区采样 + 半透明遮罩 + 实时更新）约 1-2 周

## 5. 关键技术注意点（后续设计必须覆盖）

1. 会话恢复需要区分：
- 布局恢复（窗口/项目/pane）
- 任务恢复（进程仍在 vs 已结束）
- AI 上下文恢复（会话 id、审批状态、最后建议）

2. `need approve` 状态来源必须结构化
- 优先事件总线/协议字段
- 纯文本匹配可做兜底，但稳定性较差

3. 环境变量不得明文长期落盘
- 建议接 macOS Keychain
- UI 侧做遮罩与权限提示

4. 可编辑 diff 的实质难点
- 编辑缓冲与 hunk 映射同步
- 增量重算与大文件性能控制

5. 背景图自动适配的关键点
- 需要建立稳定的可读性算法（建议基于 WCAG 对比度阈值）
- 避免频繁抖动变色（加入滞回/阈值缓冲）
- 大图采样与模糊处理要控性能，避免影响渲染帧率
